# GitHub Beginner Guide (UTech Dynamics)

## 1. What is GitHub?

GitHub is a website where we store and manage project files.

We use GitHub to:

* Save documents and code
* Track changes
* Work together as a team
* Share updates in an organized way

---

## 2. How to Create a GitHub Account

### Step 1: Go to GitHub

Open: https://github.com

### Step 2: Sign Up

Click **“Sign up”**

Enter:

* Email address
* Password
* Username (choose a professional name, e.g. rohan-ud or shihon-ud)

### Step 3: Verify Email

Check your email inbox and confirm your account.

### Step 4: Complete Setup

Skip or choose basic options for now.

Now your GitHub account is ready.

---

## 3. Important GitHub Terms (Simple Meaning)

* **Repository (Repo)**: A project folder
* **Branch**: A working copy of a project
* **Commit**: Saving changes
* **Issue**: A task or problem discussion
* **Pull Request (PR)**: Asking to add your changes to the main project

---

## 4. How to Open a Repository

1. You will receive a link like:
https://github.com/UTech-Dynamics/utechdynamics-handbook

2. Click the link.

3. You will see project files and folders.

---

## 5. How to Make Changes (Simple Method)

### Option A: Edit directly on GitHub (Easiest)

1. Open a file
2. Click the ✏️ (edit) button
3. Make changes
4. Scroll down
5. Click **“Commit changes”**
6. Write a short message like:
   “Updated sales document”
7. Click confirm

---

## 6. How to Contribute to a Team Project (Pull Request Method)

This is the correct professional way.

### Step 1: Fork the repository

Click **“Fork”** button (top right)

This creates your own copy of the project.

---

### Step 2: Make changes

Open your copy and edit files.

---

### Step 3: Commit changes

Save your changes with a message.

---

### Step 4: Create Pull Request (PR)

1. Go to your repository
2. Click **“Contribute”**
3. Click **“Open Pull Request”**
4. Add a title like:
   “Added sales lead documentation”
5. Submit PR

---

### Step 5: Wait for review

The project owner will:

* Review your changes
* Suggest edits (if needed)
* Accept or reject the request

---

## 7. How We Use GitHub at UTech Dynamics

We use GitHub for:

* Company documents
* Sales scripts
* Product descriptions
* Training materials
* Process improvements

Not only for coding.

---

## 8. Rules

* Do not delete files without permission
* Do not make random changes
* Always write clear commit messages
* Ask before making major edits
* Be professional in comments

---

## 9. Your First Task

After creating your GitHub account:

1. Share your username
2. Access the UTech Dynamics repository
3. Read the onboarding document
4. Write a short summary in Issues
